Bem-vindo à Eorzea, aventureiro. Antes de empunhar qualquer arma, você precisa
entender a Cidade-Estado para onde acabou de chegar. Este capítulo é a sua
orientação na Guilda dos Aventureiros: o que esperar deste tomo, em que ritmo
percorrê-lo, e o que fazer quando travar.

## O que este guia é

Um caminho do nível 1 (nunca programou) até o seu primeiro estágio em **Python**.
Nada além disso. Não é referência, não é livro de algoritmos, não é roadmap de
LinkedIn. É o caminho.

A jornada está dividida em cinco **Tomos**:

- **Tomo 0 (Main Story Quest)**: o que fazer antes de escrever código. Instalar
  Python, abrir o terminal, criar conta no GitHub. Você está aqui.
- **Tomo I (Sua primeira extreme)**: variáveis, condicionais, loops, funções.
  As mecânicas básicas que toda magia em Python usa.
- **Tomo II (Savage)**: orientação a objetos, APIs, banco de dados, testes.
  As mecânicas avançadas que separam quem brinca de quem trabalha.
- **Tomo III (Farm de BIS)**: cinco mini-projetos no formato de Trial. Cada um
  vira um repositório no seu GitHub.
- **Tomo IV (Ultimate cleanup)**: o caminho até o estágio. GitHub, currículo,
  LinkedIn, entrevista.

Cada Tomo abre o próximo. Não pule, mesmo que algo pareça simples demais.
ARR existe por uma razão.

## Em que ritmo

A regra simples: **um capítulo por dia, com o exercício no fim**. Cinco dias
por semana. Dois dias para descansar (sério, descansar é parte do treino).

Isso significa que o Tomo 0 cabe em uma semana. Tomo I cabe em três. Tomo II
em duas. Tomo III leva mais, porque cada Trial é maior. Tomo IV em uma.

No total: dois meses se você for consistente. Dois anos se você for inconsistente.
A diferença é literalmente toda nos sete dias da semana.

:::tip[Dica de raid]
Se você sentir que um capítulo está fácil, faça os exercícios mesmo assim.
Quem pula mecânica simples na ARR morre tomando hit básico em Stormblood.
:::

## Como ler cada capítulo

Cada capítulo segue a mesma estrutura:

1. **Abertura**: contextualiza o assunto e diz pra que serve no mundo real.
2. **Conteúdo**: a parte técnica, com exemplos de código que você deve digitar
   à mão (não copiar e colar). Digitar grava na memória de jeito diferente.
3. **Exercícios**: pequenos desafios. Você deve fazer todos antes do próximo
   capítulo. Sem exceção.
4. **Você concluiu**: o resumo do que você é capaz de fazer agora.

Faça o exercício mesmo se for óbvio. Especialmente se for óbvio.

## O que fazer quando travar

Você vai travar. É parte do trabalho. Quando travar:

1. **Leia a mensagem de erro inteira**, de baixo pra cima. Quase sempre a linha
   culpada está lá, com nome de arquivo e número.
2. **Pesquise o erro literal no Google**, copiando a última linha do traceback.
   Stack Overflow tem 90% das respostas que você precisa.
3. **Mude uma coisa por vez** quando estiver tentando consertar. Se mudar duas
   e o programa funcionar, você não sabe qual delas resolveu.
4. **Vá dormir**. Sério. 80% dos bugs que parecem impossíveis às 23h são
   ridículos às 7h da manhã.
5. **Me chama** se nada disso resolver. Mas faça as quatro primeiras antes.

## O portfólio começa hoje

Cada exercício, cada Trial, cada anotação que você fizer ao longo do guia
deve ir para um repositório no seu GitHub. Pode parecer cedo demais para
começar a publicar coisas, e é isso o ponto: quando você chegar no Tomo IV
e for montar o currículo, vai ter um histórico real de aprendizado pra
mostrar. Recrutador adora ver progresso.

Crie hoje um repositório chamado `aprendendo-python` (ou o nome que você
preferir). A cada exercício que terminar, adicione, faça commit, e dê push.
Você vai aprender a usar Git no último capítulo deste Tomo, então por
enquanto pode usar o site do GitHub direto se ficar mais fácil.
